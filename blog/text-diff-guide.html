What is Text Diff?

Text diff (short for "difference") is the process of identifying changes between two text blocks. A diff tool highlights what was added, removed, or modified — making it easy to spot changes at a glance.

Common Use Cases

  • Code review: Compare code changes before committing or merging PRs
  • Document comparison: Track revisions in articles, reports, or legal documents
  • Configuration management: Spot changes in config files between environments
  • Data validation: Compare CSV exports, logs, or API responses
  • Plagiarism checking: Quickly spot similarities or differences between texts

How Diff Algorithms Work

Line-by-Line Diff

The most common approach compares texts line by line. The algorithm finds the Longest Common Subsequence (LCS) — the longest sequence of unchanged lines — and marks everything else as additions or removals. This is the default mode for most diff tools.

Character-Level Diff

For changed lines, character-level diff drills down to show exactly which characters within a line changed. This provides a finer-grained view than line-level diff alone.

Frequently Asked Questions

What's the difference between unified and side-by-side diff?

Side-by-side diff shows original and modified text in two columns, making it easy to scan. Unified diff combines both versions into a single view with markers for added/removed lines. Our tool supports both.
